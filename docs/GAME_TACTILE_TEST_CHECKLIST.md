# Dot Forest 게임 촉각 접근성 테스트

## 독립 조작

- [ ] 화면을 보지 않고 타이틀에서 `Tab`으로 독립 시작을 선택할 수 있다.
- [ ] F1으로 현재 구역, 대략적 위치, 바라보는 방향을 확인할 수 있다.
- [ ] 패닝키로 이동하고 성공·실패를 구분할 수 있다.
- [ ] 막힌 이동은 막힌 이유와 다음 추천 방향을 알려준다.
- [ ] F2 또는 Enter/Space로 도트링을 수집하고 대화 선택을 확정할 수 있다.
- [ ] F3으로 현재 미션을 확인할 수 있다.
- [ ] F4로 주변 위험, 목표 방향, 상호작용 대상과 촉각 범례를 다시 확인할 수 있다.
- [ ] PanningAll으로 현재 주변, 전체 맵, 목표 방향 보기를 전환할 수 있다.
- [ ] Esc로 열린 안내·메뉴를 닫을 수 있다.
- [ ] 설정에서 게임을 다시 시작할 수 있다.

## DotPad 촉각

- [ ] 플레이어 2×2 강한 블록을 즉시 찾을 수 있다.
- [ ] 플레이어 방향 꼬리가 현재 진행 방향과 일치한다.
- [ ] 목표 다이아몬드와 위험 X 패턴이 혼동되지 않는다.
- [ ] 벽의 굵은 선과 이동 가능한 빈 공간이 구분된다.
- [ ] 상호작용 대상의 3점 클러스터가 목표·위험과 구분된다.
- [ ] 한 화면의 핵심 객체가 5개를 넘지 않는다.
- [ ] 한 화면의 패턴 종류가 5개를 넘지 않는다.
- [ ] 이동 후 플레이어 위치 변화가 250ms 안에 전송된다.
- [ ] 동일한 촉각 프레임은 중복 전송되지 않는다.

## 세 가지 보기 모드

- [ ] 현재 주변 보기에서 플레이어가 중앙 부근에 있고 가까운 위험이 우선 표시된다.
- [ ] 전체 맵 보기에서 플레이어, 목표, 주요 벽, 출구만 단순화해 표시된다.
- [ ] 목표 방향 보기에서 플레이어부터 목표까지 점선 방향선이 표시된다.
- [ ] 화면 밖 목표는 가장자리 방향 마커로 표시된다.

## TTS와 피드백

- [ ] 위치 설명은 한 번에 이해할 수 있을 만큼 짧다.
- [ ] 주변 스캔은 구역, 목표, 위험, 추천 행동 순서로 읽는다.
- [ ] 이동 성공은 목표에 가까워졌는지 또는 멀어졌는지 알려준다.
- [ ] 수집 실패와 이동 실패는 이유를 알려준다.
- [ ] 위험·목표·일반 이동 안내가 과도하게 겹치지 않는다.

## 완료 테스트

- [ ] 화면을 보지 않고 첫 구역의 목표를 찾는다.
- [ ] 도트링 또는 미션 대상을 구분한다.
- [ ] 장애물을 피해 이동한다.
- [ ] 상호작용을 완료한다.
- [ ] 다음 구역으로 이동한다.
